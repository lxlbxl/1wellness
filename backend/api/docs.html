Introduction

The 1wellness API is a bridge designed to connect your database with automation tools like n8n, Zapier, or Make. It supports retrieving, filtering, updating, and deleting records from your core tables.

Base URL: https://yourdomain.com/n8n-export.php

Authentication

All requests must be authenticated using the API key defined in your server configuration.

X-API-KEY: your_secret_api_key

GET Users & Leads

Retrieve a list of users. This endpoint includes computed fields to quickly identify customer value.

GET /n8n-export.php?type=users

Parameters

Parameter Description
email Full email or wildcard (e.g., *@gmail.com).
phone New Full phone or wildcard (e.g., *1234).
exclude_buyers New If 1, removes anyone who is present in the sales table.

Sample Output

{
    "id": 101,
    "first_name": "Alice",
    "email": "alice@example.com",
    "is_buyer": 1  // Computed automatically
}

GET Sales Data

Retrieve sales transactions. Supports filtering by date range and payment status.

GET /n8n-export.php?type=sales

GET Assessments

Retrieve health assessments submitted by users. You can fetch specific categories or all assessments.

GET /n8n-export.php?type=pcos_assessments

Supported Assessment Types

Use these values for the type parameter:

  • assessments (Generic/All)
  • pcos_assessments
  • acne_assessments
  • weight_assessments

Specific Filters

assessment_type Filter generic assessments table by type (e.g., 'pcos', 'acne').
include_user If 1, includes the full user profile who took the assessment.

JSON Parsing

Fields like symptoms, lifestyle_factors, and diet_preferences are stored as JSON strings in the database but are automatically parsed into objects in the API response.

Sample Output

{
    "success": true,
    "data": [
        {
            "id": 50,
            "type": "pcos_assessments",
            "symptoms": {
                "irregular_periods": true,
                "acne_severity": "moderate"
            },
            "diet_preferences": ["gluten_free", "dairy_free"],
            "created_at": "2023-10-15..."
        }
    ]
}

GET Single Record

Fetch a specific record by its unique ID.

GET /n8n-export.php?type=sales&id=550

Filtering & Search Examples

1. Get "Pure" Leads (Exclude Buyers)

curl -H "X-API-KEY: 123" "https://api.com/n8n-export.php?type=users&exclude_buyers=1"

2. Search by Phone Number

curl -H "X-API-KEY: 123" "https://api.com/n8n-export.php?type=users&phone=*5678"

3. Get PCOS Assessments with User Data

curl -H "X-API-KEY: 123" "https://api.com/n8n-export.php?type=pcos_assessments&include_user=1"

POST Update Record

Modify an existing record. Send a JSON body with the action: "update" parameter.

POST /n8n-export.php

JSON Body

{
    "action": "update",
    "type": "users",
    "id": 105,
    "data": {
        "status": "active",
        "first_name": "Jane"
    }
}

POST Delete Record

POST /n8n-export.php

JSON Body

{
    "action": "delete",
    "type": "contacts",
    "id": 404
}

Incremental Sync

To process only "new" records in n8n, use the since_id parameter.

GET /n8n-export.php?type=sales&since_id=1050
